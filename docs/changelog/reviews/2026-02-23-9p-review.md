## Fix Details

### BUG-9P-9 — Timeout cap in `handle9PConnect`
**Before:** `const { host, port = 564, timeout = 10000 } = body;` — raw user value passed through.
**After:** `const cappedTimeout = Math.min(timeout, 30000);` — capped at 30s, consistent with other handlers.

### BUG-9P-10 — `read9PMessage` max-size guard
**Before:** Loop accumulated bytes until `buf.length >= size` with no cap on `size`.
**After:** Added `if (size > 65600) break;` after reading the 4-byte size field. 65600 = 65536 (max user-requested read) + 64 bytes of message overhead. Any announced size larger than this is rejected immediately.

### BUG-9P-11 — Accumulating reader in `handle9PConnect`
**Before:** Both Rversion and Rattach read via `readFromSocket(reader, timeoutPromise)` — a single `reader.read()` call.
**After:** Both use `read9PMessage(reader, timeLeft())`, which accumulates chunks until it has a complete message of the size announced by the 4-byte header.

### BUG-9P-12 — POST method check
**Before:** `handle9PConnect` had no method guard.
**After:** Added `if (request.method !== 'POST') return 405 JSON error` at the top of the function, matching the other handlers.

### BUG-9P-13 — Sub-operation timeouts
**Before:** All three handlers called `ninePHandshake(reader, writer, timeout)` then used `read9PMessage(reader, 5000)` / `read9PMessage(reader, 8000)` for subsequent operations.
**After:** Each handler now records `connectTime = Date.now()` before opening the socket and defines `timeLeft = () => Math.max(timeout - (Date.now() - connectTime), 1000)`. All `read9PMessage` calls and the `ninePHandshake` call use `timeLeft()` so the total wall-clock time across all operations is bounded by the user-specified timeout.

### BUG-9P-14 — Root fid Tclunk
**Before:** `handle9PStat` and `handle9PRead` only sent `Tclunk` for walked/opened fids, leaving rootFid=1 open.
**After:**
- `handle9PStat` root path: adds `Tclunk(rootFid)` with tag 3 after the stat.
- `handle9PStat` non-root path: adds `Tclunk(rootFid)` with tag 5 after clunking targetFid (tag 4).
- `handle9PRead`: adds `Tclunk(rootFid)` with tag 6 after clunking fileFid (tag 5).
- `handle9PLs`: adds `Tclunk(rootFid)` with tag 6 when `dirFid !== rootFid` (non-root walk).

### BUG-9P-15 — UI: stat/read/ls operations
**Before:** `NinePClient.tsx` had a single connect form with no way to reach stat/read/ls.
**After:** Added tab selector (`connect | stat | read | ls`) with:
- **Stat**: host + port + optional path → shows name, mode string (`drwxr-xr-x`), size, uid/gid, timestamps, QID
- **Read**: host + port + required path + optional offset/count → decodes content as text if printable, falls back to raw base64; shows bytesRead
- **ls**: host + port + optional path → tabular listing with mode, size, mtime, filename for every entry

Also added `describeQidType()` helper and `formatMode()` / `formatTimestamp()` utilities for human-readable display.

### BUG-9P-16 — QID type bitfield
**Before:** `data.rootQid.type === 0x80 ? 'directory' : data.rootQid.type === 0 ? 'file' : hex`
**After:** `describeQidType(t)` uses bitfield tests (`t & 0x80`, `t & 0x04`, etc.) so composite types (e.g. `0xC0` = directory+append) report correctly.

## Fix Details

### BUG-9P-19 — `buildTread` 64-bit offset support
**Before:** `view.setUint32(4, offset, true); view.setUint32(8, 0, true);` — high 32 bits always 0.
**After:** Splits offset into low/high 32-bit parts: `offsetLow = offset >>> 0; offsetHigh = Math.floor(offset / 0x100000000) >>> 0;` — supports offsets up to Number.MAX_SAFE_INTEGER (2^53).

### BUG-9P-20 — Rwalk nwqid validation
**Before:** All three handlers (stat, read, ls) checked `rw.type === Rwalk` but never parsed the Rwalk body.
**After:** Each handler now parses `nwqid` from the Rwalk body (`uint16LE` at offset 0) and throws `Walk incomplete: server walked N of M components` if `nwqid < pathComponents.length`. Applied to `handle9PStat`, `handle9PRead`, and `handle9PLs`.

### BUG-9P-21 — Connect handler Tclunk
**Before:** `handle9PConnect` used fid=0 for Tattach but never sent Tclunk before closing.
**After:** Sends `Tclunk(fid=0)` after a successful Rattach, wrapped in try/catch like the other handlers.

### BUG-9P-22 — Composite QID type flags
**Before:** `describeQidType` used `if/return` chain — `0xC0` showed only "directory".
**After:** Collects all matching flags into an array and joins: `0xC0` → "directory, append-only". Falls back to "file" if no flags match.

### BUG-9P-23 — Plan 9 mode bits in formatMode
**Before:** Only showed `d` (directory) or `-` (file) prefix with Unix rwx bits.
**After:** Shows `A` for auth files, and appends `[a]`/`[l]`/`[t]` flags for append-only, exclusive, and temporary files. Example: `drwxr-xr-x [a]` for an append-only directory.

### BUG-9P-24 — Truncated directory indicator
**Before:** `/api/9p/ls` returned entries with no indication that the listing might be incomplete.
**After:** Response now includes `truncated: true` when `bytesRead >= dirReadCount` (the read returned as many bytes as requested, suggesting more data exists). UI shows "(listing may be truncated — directory exceeds single read)" next to the entry count.

### BUG-9P-25 — Documentation timeout correction
**Before:** NINEP.md `/api/9p/connect` table: `timeout | number | 10000 | Total timeout in ms (max: no enforced limit)`.
**After:** Updated to `(max: 30000)` to match the actual `Math.min(timeout, 30000)` cap. Also updated the Security Considerations section to note all four endpoints now cap timeouts.
